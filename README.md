# PriceFinder

Android-приложение: ищет цену товара по наименованию, артикулу и модели —
**на Wildberries и Ozon одновременно**, выводит общий список предложений и
считает среднюю цену по рынку (а также минимум и максимум).
Дополнительно умеет **искать по фотографии** (распознавание на устройстве).

## Возможности
- Поиск сразу по двум маркетплейсам (Wildberries + Ozon), результаты объединяются.
- Список предложений, отсортированный по цене, с указанием источника.
- Средняя, минимальная и максимальная цена по рынку.
- Поиск по фото: камера или галерея → распознавание объекта на устройстве
  (ML Kit, бесплатно, офлайн) → автоматический поиск по распознанным меткам.

## Стек
- Kotlin + Jetpack Compose (Material 3)
- OkHttp + kotlinx.serialization, параллельные запросы к источникам (корутины)
- ML Kit Image Labeling (on-device) для распознавания фото
- MVVM (ViewModel + StateFlow)
- minSdk 24, targetSdk 34

## Как получить готовый APK (без локальной сборки)
1. Залейте репозиторий на GitHub (см. ниже).
2. Откройте вкладку **Actions** — сборка запустится автоматически.
3. Готовый `app-debug.apk` появится:
   - в артефактах сборки (**Actions → app-debug**), и
   - в разделе **Releases** (тег `latest`) — оттуда качается одним кликом.
4. Скопируйте APK на телефон и установите (разрешите установку из неизвестных
   источников).

## Локальная сборка (если есть Android SDK)

```bash
./gradlew assembleDebug
# app/build/outputs/apk/debug/app-debug.apk
```

## ⚠️ Важно про источники данных
Приложение использует **внутренние, недокументированные** эндпоинты
Wildberries (`search.wb.ru`) и Ozon (`composer-api`). Они:
- могут измениться или перестать работать без предупреждения;
- не предназначены площадками для сторонних приложений;
- их использование может нарушать правила Wildberries / Ozon.

Парсинг Ozon намеренно «защитный»: при изменении структуры ответа источник
просто вернёт пустой список (ошибка покажется в интерфейсе), а не уронит весь
поиск. Wildberries при этом продолжит работать.

Для публикации в Google Play и стабильной работы рекомендуется перейти на
официальные/партнёрские API площадок и заменить реализацию в
`data/WildberriesSource.kt` и `data/OzonSource.kt`. Остальной код
(UI, агрегатор, распознавание, статистика) менять не нужно.

### Про распознавание по фото
Используется on-device модель (ML Kit). Она определяет **тип** объекта
(«кроссовок», «часы», «сумка»), но не конкретную модель/бренд — это
ограничение бесплатных моделей на устройстве. Для точного распознавания
«до модели» нужен облачный сервис (Google Vision / специализированный API).
